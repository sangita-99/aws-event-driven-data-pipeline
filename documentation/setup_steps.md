**Setup Steps**



This section includes the steps followed to build the event-driven data processing pipeline.



*Step 1 – Create S3 Bucket*

An Amazon S3 bucket named **sales-data-takeo** was created to store the dataset and processed output.



The folder structure inside the bucket is as follows:



**sales-data-takeo**

&nbsp;	raw/

&nbsp;	processed/



The raw folder stores the original dataset while the processed folder stores the transformed output generated by the Glue job.



*Step 2 – Create SQS Queue*

An Amazon SQS queue was created to receive processing requests. The queue acts as the event trigger for the pipeline.



*Step 3 – Create AWS Lambda Function*

A Lambda function was created using Python. The function reads the message received from SQS and extracts the file information. After extracting the file path details, the Lambda function triggers an AWS Glue job using the AWS SDK (boto3).



*Step 4 – Configure Lambda Trigger*

The SQS queue was added as a trigger for the Lambda function. Whenever a new message appears in the queue, Lambda automatically executes.



*Step 5 – Create AWS Glue Job*

An AWS Glue job was created to perform the ETL transformation. The Glue script reads the CSV file from the raw folder, filters records where order status is "Completed", and writes the transformed dataset to the processed folder in Parquet format.



*Step 6 – IAM Permissions*

IAM roles were configured to allow communication between the services. Lambda was granted permission to start Glue jobs, and Glue was granted permission to read and write data from Amazon S3.
